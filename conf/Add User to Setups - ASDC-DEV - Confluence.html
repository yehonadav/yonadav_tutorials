Add User to Setups

To add new users to setups you need to insert new users into the DB and into webseal-simulator. Once you restart webseal-simulator service, you'll see your new users in the login screen. Pay notice to the fact that upgrading the setup with Test Manager will reset the users list in the webseal-simulator, so you'll need to repeat the 2nd step.

This is how webseal simulator login looks with the new QA users.

Script add_users.sh

Download the attached script and run it on the BE server as root.

The script pushes the new users, updates webseal.conf and restarts webseal-simulator service.

add_users.sh

Manual Steps

Step 0 - Login to The BE Server

The following steps are to run on the BE server.

Step 1 - Insert New Users Into The DB

Do the following as user m98835 `su - m98835`

We're using curl to add the users. In the example we're using -H 'USER_ID: jh0003' as the admin user to insert the user qa1. jh0003 doesn't need to be changed. 

When you're adding multiple users, the only unique field is  "userId": "qa1001". firstName, lastName & email don't matter.

The available roles are: Admin, Designer, Ops, Governer, Tester, Product Strategist, Product Manager.

Example

curl -i -X POST -H 'Content-Type: application/json' -H 'USER_ID: jh0003' -d '{"firstName": "Mel", "lastName": "Brooks", "email": "mb1001@asdc.com", "userId": "mb1001", "role": "DESIGNER"}' "http://localhost:8285/sdc1/feProxy/rest/v1/user"
curl -X POST -H 'Accept:application/json' -H 'USER_ID:jh0003' -H 'Content-Type:application/json' -d '{"role":"DESIGNER","status":"ACTIVE"}' "http://127.0.0.1:8285/sdc1/feProxy/v1/user/mb1001/role" | python -mjson.tool

curl -i -X POST -H 'Content-Type: application/json' -H 'USER_ID: jh0003' -d '{"firstName": "Stephen", "lastName": "Spielberg", "email": "ss1002@asdc.com", "userId": "ss1002", "role": "DESIGNER"}' "http://localhost:8285/sdc1/feProxy/rest/v1/user"
curl -X POST -H 'Accept:application/json' -H 'USER_ID:jh0003' -H 'Content-Type:application/json' -d '{"role":"DESIGNER","status":"ACTIVE"}' "http://127.0.0.1:8285/sdc1/feProxy/v1/user/ss1002/role" | python -mjson.tool

 

curl -i -X POST -H 'Content-Type: application/json' -H 'USER_ID: jh0003' -d '{"firstName": "Groucho", "lastName": "Marx", "email": "gm1003@asdc.com", "userId": "gm1003", "role": "TESTER"}' "http://localhost:8285/sdc1/feProxy/rest/v1/user"

curl -X POST -H 'Accept:application/json' -H 'USER_ID:jh0003' -H 'Content-Type:application/json' -d '{"role":"TESTER","status":"ACTIVE"}' "http://127.0.0.1:8285/sdc1/feProxy/rest/v1/user/gm1003/role" | python -mjson.tool

Step 2 - Insert new users into webseal-simulator

modify and restart webseal-simulator is also a 2 step task.

Modify webseal.conf

Do the following as user m98835.

To add the users into Webseal-Simulator you need to modify the file /data/config/sdc-simulator/webseal.conf  (for deployments before May-01: /apps/jetty/base/webseal-simulator/config/webseal.conf)


add the section:

{
userId="mb1001"
password="123123a"
firstName="Mel"
lastName="Brooks"
role="Designer"
email="mb1001@asdc.com"
},

{
userId="ss1002"
password="123123a"
firstName="Stephen"
lastName="Spielberg"
role="Designer"
email="ss1002@asdc.com"
},

{
userId="gm1003"
password="123123a"
firstName="Groucho"
lastName="Marx"
role="Tester"
email="gm1003@asdc.com"
}

so it looks like in the screenshot below.

Restart webseal-simulator Service

Do the following as user root `ctrl+d`

run the command `service webseal-simulator restart`

Screenshot of webseal.conf

Change User Role

To change the user role:

  1. Run the 2nd curl command.
  2. Modify webseal-conf and restart webseal-simulator service.
  3. Clear your browser cookies.
  4. Re-login from http://YOUR_FE_SERVER:8285/login

Example

Run Curl Command

as user m98835

curl -X POST -H 'Accept:application/json' -H 'USER_ID:jh0003' -H 'Content-Type:application/json' -d '{"role":"ADMIN","status":"ACTIVE"}' "http://127.0.0.1:8285/sdc1/feProxy/rest/v1/user/qa1001/role" | python -mjson.tool

Modify webseal-simulator

{
userId="qa1001"
password="123123a"
firstName="QA1"
lastName="QA1_surname"
role="Admin"
email="qa1@asdc.com"
}

 

Multi Servers Deployment

When adding users on a multi-servers deployment (Full Deployment):

  1. The add users curl commands need to be run on the BE server.
  2. The webseal-simulatore/webseal.conf file is edited on the FE server.

 Older ASDC Releases Setup

The below is valid for older ASDC releases setup.

In order to add you ATT user to a setup please follow this steps:

  1. SSH login to the BE using the listed credentials: AMDOCS and AT&T ASDC Integration Setups

  2. Execute the following REST with you ATT UID:


    ?
    curl -X POST -H 'Accept:application/json' -H 'HTTP_CSP_ATTUID: jh0003' -H 'Content-Type: application/json' -d '{"firstName":"","lastName":"","attuid":"az2497","email":"az2497@att.com","role":"DESIGNER"}' "http://127.0.0.1:8285/sdc1/feProxy/rest/v1/user" | python -mjson.tool
